## 1. Product Overview

### 1.1 Vision
BrandForge AI is an autonomous brand intelligence and marketing asset generator. A user pastes any company URL and receives a complete, downloadable brand kit — flyers, social media cards, email templates, ad copy, and competitive insights — generated by a multi-agent AI pipeline that deeply understands the brand first.

### 1.2 Problem Statement
Startups, freelancers, and small businesses lack the budget for marketing agencies and the time to manually produce consistent brand assets. Existing AI tools (ChatGPT, Canva AI) are generic — they don't *know* your brand. BrandForge reads your brand, internalizes it, then generates assets that are on-tone, on-color, and on-message.

### 1.3 Target Users (for scope clarity)
- **Primary demo persona:** A startup founder who pastes their URL and gets a brand kit in 3 minutes
- **Portfolio persona:** An AI systems evaluator who sees multi-agent orchestration, RAG, and real output

### 1.4 Success Metrics
- End-to-end pipeline completes in < 3 minutes for a typical site
- Generated flyer passes brand color consistency check (automated)
- Critic agent catches > 70% of tone-inconsistent copy (demo-able)
- UI renders live agent progress (SSE streaming working)
- One-click ZIP download of full brand kit

## 2. Core Features (MVP Scope)

### F1 — URL Ingestion & Brand Scraping
- User inputs a company URL
- System scrapes homepage + up to 5 sub-pages (about, products, pricing, blog, contact)
- Extracts: brand name, tagline, product list, USPs, tone descriptors, color palette, target audience signals
- Fallback chain: Jina Reader API → Playwright → BeautifulSoup

### F2 — Brand Intelligence RAG
- All scraped content chunked and embedded via `sentence-transformers/all-MiniLM-L6-v2`
- Stored in ChromaDB (local persistent store)
- RAG queries power every downstream agent: "What problems does this brand solve?", "What is their brand tone?"
- Brand profile object stored as structured JSON (colors, tone, audience, USPs)

### F3 — Multi-Agent Creative Pipeline (LangGraph)
Four agents running in parallel after brand extraction:
- **Copywriter Agent:** Headlines, taglines, email subject lines, CTA variations, ad copy (3 tones: bold / friendly / professional)
- **Layout Agent:** Determines optimal flyer structure for brand category (SaaS / retail / NGO / food / health)
- **Email Agent:** Full email campaign — welcome email, promotional email, re-engagement email
- **Ad Agent:** Google ad copy (headlines + descriptions), Instagram caption variants, LinkedIn post

All four feed into a **Critic Agent** (fan-in) that reviews tone consistency and brand alignment before asset generation.

### F4 — Asset Generation
- **Flyers:** HTML/CSS template rendered to PDF via Playwright `.pdf()` — brand colors auto-injected
- **Social Cards:** Pillow-generated PNG images (1080×1080) with brand palette + generated copy
- **Email Templates:** Styled HTML files ready to drop into Mailchimp/Brevo
- **Ad Copy Doc:** Markdown + PDF export of all ad variations

### F5 — Brand Kit Export
- All assets bundled into a ZIP file
- Downloadable via the UI with one click
- Includes: `brand_profile.json`, `flyer.pdf`, `social_card.png`, `email_welcome.html`, `ad_copy.pdf`

### F6 — Live Agent Progress UI
- Server-Sent Events (SSE) stream agent status updates to the frontend in real time
- User sees: "Scraping site... → Extracting brand colors... → Copywriter Agent running... → Generating flyer..."
- Not a spinner — a live activity feed with agent names and statuses

## 5. User Stories

### Core Flow
- **US-01:** As a user, I can paste a URL and click "Forge Brand Kit" to start the pipeline
- **US-02:** As a user, I can watch live agent progress so I know the system is working
- **US-03:** As a user, I receive a brand profile summary (colors, tone, audience) before assets are ready
- **US-04:** As a user, I can preview each generated asset in the UI before downloading
- **US-05:** As a user, I can download all assets as a single ZIP file

### Power Features
- **US-06:** As a user, I can regenerate any single asset (e.g. just the flyer) without re-running the full pipeline
- **US-07:** As a user, I can see the extracted brand color palette visually displayed
- **US-08:** As a user, I can switch copy tone (bold / friendly / professional) and regenerate copy

## 7. API Contract (Backend ↔ Frontend)

### POST `/api/forge`
**Request:**
```json
{ "url": "https://example.com" }
```
**Response:**
```json
{ "job_id": "uuid-1234" }
```

### GET `/api/forge/{job_id}/stream` (SSE)
Streams newline-delimited JSON events:
```json
{ "stage": "scraping", "status": "running", "message": "Fetching homepage..." }
{ "stage": "brand_extraction", "status": "complete", "data": { "colors": [...], "tone": "..." } }
{ "stage": "copywriter_agent", "status": "running", "message": "Generating headlines..." }
{ "stage": "complete", "status": "complete", "download_url": "/api/forge/uuid-1234/download" }
```

### GET `/api/forge/{job_id}/result`
Returns full brand kit JSON (brand profile + all copy variants).

### GET `/api/forge/{job_id}/download`
Returns ZIP file as binary download.

## 8. Risk Register

| Risk | Likelihood | Mitigation |
|---|---|---|
| Site blocks Jina Reader | Medium | Playwright fallback; cache results |
| Groq rate limit hit | Medium | Gemini fallback; add delay between calls |
| Playwright PDF CSS issues | Low | Maintain 2-3 tested HTML templates |
| ChromaDB corruption on crash | Low | Wrap ingest in try/except; re-ingest on failure |
| LLM hallucinating brand colors | Medium | Always extract colors from CSS/HTML, never from LLM |

## 9. Glossary

| Term | Definition |
|---|---|
| Brand Kit | The complete set of generated assets for one URL |
| Brand Profile | Structured JSON of extracted brand intelligence |
| Critic Agent | LangGraph node that reviews all copy before asset generation |
| Fan-out / Fan-in | LangGraph pattern: parallel agent execution → single aggregation node |
| SSE | Server-Sent Events — one-way real-time stream from server to browser |
